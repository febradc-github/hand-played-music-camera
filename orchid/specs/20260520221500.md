# Hand-Tracked Chord Instrument
Date: 20260520221500
Status: approved

## Problem
Play chords by pointing index fingers at radial menus on screen — left finger picks root note (A-G), right finger picks modifier (#, b, m, aug, add9, etc.), combined into a MIDI chord with sustained output.

## Approach
PyGame UI (modular): MediaPipe tracks fingers from USB webcam in a background thread. PyGame renders two radial pie menus in a dedicated window with highlight-on-select feedback. `python-rtmidi` sends MIDI note-on/note-off for sustained chord playback.

## Key Decisions
- MediaPipe Hands for fingertip landmark detection (index finger tip = landmark 8)
- Two radial menus drawn with PyGame: left circle = A-G root notes (7 segments), right circle = chord modifiers (segments TBD)
- Left index finger (landmark 8) mapped to root note zone, right index finger to modifier zone
- Chord triggered when both fingers are in valid zones simultaneously
- MIDI note-on sent on chord change, note-off sent when either finger leaves its zone (sustained while held)
- `python-rtmidi` over mido — lower latency for real-time performance
- Background thread for camera capture + MediaPipe inference, main thread for PyGame rendering + MIDI output
- USB webcam accessed via OpenCV (`cv2.VideoCapture`)
- Configuration file for chord modifier list and MIDI output port

## Trade-offs Accepted
- PyGame visuals are functional but not GPU-accelerated — acceptable for radial menus at this scale
- Thread communication adds ~1 frame latency vs single-threaded OpenCV approach
- MediaPipe Hand model expects reasonable lighting and hand visibility

## Out of Scope
- Multi-camera support
- Hand gestures beyond index finger pointing
- Recording/playback
- GUI configuration editor (use config file)
- Audio synthesis (MIDI only)
