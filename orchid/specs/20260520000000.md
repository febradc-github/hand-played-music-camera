# Index Finger Calibration Phase
Date: 20260520000000
Status: approved

## Problem
No finger baseline — absolute fingertip position drives menus, requiring precise physical hand placement. Add a calibration phase that captures neutral index-finger positions so the main loop uses relative delta tracking (touchpad-style).

## Approach
**Cursor-style delta tracking with recalibratable baselines.** A calibration phase presents two boxes (left/right) for the user to hold index fingers in for 3 seconds. This captures baseline (x, y) per hand. In the main loop, finger deltas from baseline drive a visible cursor on each radial menu. Recalibrate anytime via Space key.

## Key Decisions
- Two calibration boxes: left at normalized (0.25, 0.5), right at (0.75, 0.5), each 100px radius, labeled "Left Index" / "Right Index"
- 3-second continuous hold required per box; progress bar shown per side; timer resets if finger leaves box
- Both boxes must stabilize simultaneously for the timer to advance (state machine: IDLE -> ONE_PRESENT -> BOTH_PRESENT -> HOLDING -> DONE)
- Baseline stored as normalized (x, y) per hand in a `CalibrationResult` dataclass
- Main loop converts fingertip to cursor: `cursor_x = menu_center_x + (finger_x - baseline_x) * sensitivity`, clamped to screen; sensitivity defaults to 2.0
- Cursor rendered as a small filled circle (radius 8px, green) on each menu
- Existing `_get_segment` / `_compute_segment_index` reused with cursor coords instead of raw fingertip coords
- Space key triggers recalibration from main loop at any time
- New module: `src/calibrator.py` with `CalibrationResult` dataclass, `CalibrationState` enum, and a `run_calibration(tracker, renderer)` generator/coroutine
- Renderer gains `draw_calibration(state, left_present, right_present, progress)` and `draw_cursor(menu_side, x, y)` methods

## Trade-offs Accepted
- Calibration adds startup friction (~3 seconds) but eliminates need for precise hand positioning
- Cursor sensitivity is a hardcoded constant (2.0) — no user-facing config yet; config file toggle can be added later
- Calibration boxes at fixed normalized positions — won't work well at extreme camera angles
- No cursor clamping to menu radius — cursor can exit the circle; chord only triggers when cursor lands in a wedge (same as current behavior for finger-outside-menu = no chord)

## Out of Scope
- Visual sensitivity adjustment UI
- Per-user calibration profiles
- Handedness flipping (left-handed mode)
- Single-hand mode (both boxes required)
